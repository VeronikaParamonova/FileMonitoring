# FileMonitoring
1. Постановка Задачи
  В рамках первой лабораторной требуется разработать консольное приложение для отслеживания изменений состояния текстовый файлов (формат txt) с помощью сигнально-слотовой системы. Предлагается отслеживать следующие состояния исходя из характеристик размера и существования:
1. Файл существует , файл не  пустой - на экран выводится факт существования файла и его  размер.
2. Файл существует, файл был изменен - на экран выводится факт существования файла, сообщение о том что файл был изменен и его размер.  
3. Файл не существует - на экран выводится информация о том что файл не существует.
Необходимо организовать ввод/вывод, а также корректный выход из приложения

2. Предлагаемое решение
  В решении предлагается реализовать менеджер, который и будет отслеживать изменения за счёт постоянного "опроса" необходимых файлов.
В решении для менеджера используется Паттерн "Одиночка". Для всего приложения в целом соблюдаются такие принципы SOLID как: Принцип единственной ответственности, принцип открытости закрытости. Хочется отметить, что некоторые нарушения принципов допускаются для отсутствия утяжеления архитектуры.
Uml диаграмма прилагается письменно.
   Предлагаемые сущности:
   FileManager - главная сущность, выполненная по паттерну "Одиночка". Его задача - отслеживание изменений в размере и факте существования подключённых к нему файлов раз в такт таймера. Он выполняет задачи подключения/отключения, "опрос" файлов, отправка сигналов на логирование, реагирование на изменение характеристик файла
   FileIsWatched - вторая по важности сущность. Выполняет роль "обёртки" файла для хранения его характеристик. Таким образом, для каждого отслеживаемого файла создаеётся объект описываемого класса. При "опросе" менеджер вызывает у каждого файла метод проверки состояния. В случае изменения состояния FileIsWatched испускает сигнал для ьенеджера
   LogToConsole - логгер для вывода всех сообщений: об ошибках, об изменении состояния, о начале работы программы и т.д. (одно из нарушений принципа единственной ответственности для облегчения архитектуры)
   InputHandler - обработчик входных данных. Предназначен для проверки входных строк на ошибки и приведения к общему виду (удаление пробелов)

3. Инструкция пользователя
  Для запуска консоли в ОС Windows нажмите Win+R (можно использовать меню Пуск), в появившейся строке наберите "cmd" и нажмите Enter. 
В консоли необходимо перейти в директорию, которая хранит исполняемый файл (обычно это папка debug внутри build), командой cd путь/к/директории
Далее необходимо запустить программу название.exe
После появления сообщения о старте работы начинайте использование.
Для подключения файла необходимо ввести путь к файлу, который требуется отслеживать, вместе с форматом. ВНИМАНИЕ! Приложение предназначено для работы только с файлами формата txt, в противном случае будет выведено уведомление об ошибке неправильного формата. 
Пути необходимо вводить-подключать по одному
Для отключения файла из-под наблюдения введите его путь повторно.
Для отслеживания файла, который находится в одной папке с исполняемым файлом, достаточно ввести его имя вместе с форматом. Выведется соответствующее предупреждение.
Для выхода из программы введите ключевое слово EXIT, обратите внимание, регистр учитывается
